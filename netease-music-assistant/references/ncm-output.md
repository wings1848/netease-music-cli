# 输出格式

## 强制要求
1. 推荐"歌单"或"单曲"必须输出可点击链接
   - 歌单：`https://music.163.com/#/playlist?id=<明文originalId>`
   - 单曲：`https://music.163.com/#/song?id=<明文originalId>`
2. **ID 必须使用明文 originalId（数字）**，禁止加密 ID
3. 每条输出"评分 + 两层理由"（40～80字）
4. 有 `coverImgUrl` 时建议输出 `🖼️ <url>`

## 标准格式

```
🎵 音乐推荐 · [YYYY-MM-DD HH:MM]

[触发来源描述，如：周末夜晚推送]

① 🎶 [歌单/专辑/单曲名]
   ⭐ 评分：[0-100]
   📝 理由：[两层推荐理由（20～60字）]
   🖼️ [封面链接（可选）]
   🔗 [资源链接]

② 🎶 [名称]
   ...

共 N 条
```

### 链接格式（统一）

> **ID 必须使用明文 originalId（数字）**

```
https://music.163.com/#/song?id=<明文ID>
https://music.163.com/#/playlist?id=<明文ID>
https://music.163.com/#/album?id=<明文ID>
https://music.163.com/#/artist?id=<明文ID>
```

### 评分格式
```
⭐ 评分：<0-100>
```
模型自评，表示与"用户需求 + 偏好画像"的匹配度。

### 两层理由格式

**必须包含两层**（合计 40～80 字）：

1. **偏好关联层**：引用偏好证据（用户红心高频曲风/艺人/内容标签/近期趋势/常听时段）
2. **内容特质层**：描述歌单/专辑/单曲本身的氛围与适配场景

禁止：只写空泛形容词（如"很适合你"），必须落到具体证据与具体特质。

### 飞书渠道且推荐结果包含 `coverImgUrl` 时推送封面图片

1. 在飞书渠道可选发送图片消息（图片为 `coverImgUrl`）
2. 文字消息里保留 `🖼️ <coverImgUrl>` 作为兜底

若图片发送失败，不影响推荐正文；保证链接输出。

推荐内容以单曲为主时，主动询问用户是否创建歌单：
1. 使用 ncm-cli：创建歌单 → 添加歌曲
2. 验证 trackCount 确认写入成功
3. 在推荐输出中附上歌单链接
